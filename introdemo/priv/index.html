Ostracon Visualization

Basic usage of the client-side OstraconJS library to communicate with the Ostracon runtime located on the same server using an HTML5 canvas.

About Ostracon

  • Distributed voting system
  • Votes received as keystrokes
  • Voting intervals defined by callback module
  • Originally inspired by Twitch Plays Pokemon

Original Callback Module Ideas

  • Maze game with multiple clients per player
  • Final Fantasy style game with teams
  • Distributed Snake

Design and Architecture


Problems and Development

  • Difficulty in maintaining consistent state and votes
    • By default, Erlang processes don't share memory
    • Message passing infeasible due to quick clock cycles
    • Using SQL/Mongo DB added unneeded complexity
    • ETS structure solved all these problems
  • Originally decided on SockJS for websockets
    • Only known option for our websockets initially
    • Later discovered Cowboy could take care of it all
